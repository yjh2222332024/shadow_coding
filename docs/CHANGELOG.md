# 变更日志

本文件记录 Shadow_Coding 的所有重要变更。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [3.2.2] - 2026-03-25

### 🛡️ 安全修复

- **修复 ProtectedDict 盐值可预测问题** - 使用 `secrets.token_hex(32)` 替代 `random.random()`
- **增强常量折叠引擎** - 支持 f-string、format、join、bytes.decode 的静态分析
- **递归别名追踪** - 支持 5 层深度的 import 别名解析
- **装饰器审计** - 拦截使用危险函数作为装饰器的攻击
- **路径边界校验** - 严格检查所有文件操作不超出允许目录
- **原子化写入** - 使用临时文件 + 重命名消除竞态条件
- **日志脱敏** - 自动隐藏日志中的绝对路径和用户信息

### 🛠️ 技术改进

- 添加 `SecurityViolation` 作为 `SecurityError` 的别名，保持向后兼容
- 优化 `_get_base_name` 方法，正确处理属性访问链
- 扩展 `DANGEROUS_CALLS` 列表，添加 importlib 和 socket 相关函数
- 改进 `SecureLogger` 支持 Windows 和 Linux 路径脱敏

### 🧪 测试

- 完善安全实验室测试套件
- 添加 AST 混淆绕过测试
- 添加别名追踪测试
- 添加动态注入测试
- 添加路径边界测试
- 添加内存保护测试

## [3.2.0] - 2026-03-23

### ✨ 新功能

- **自适应噪声注入** - 基于变量敏感度的分级噪声
  - 高敏感变量（balance, password）→ 50% 噪声
  - 低敏感变量（temp, result）→ 10% 噪声
- **多 API Key 支持** - 跨厂商分发逻辑分片
- **安全日志系统** - logging 模块集成

### 🛡️ 安全增强

- 逻辑碎片化引擎 V4

- 递归复杂度计算

## 版本说明

### 语义化版本

- **主版本号 (Major)** - 不兼容的 API 变更
- **次版本号 (Minor)** - 向后兼容的功能新增
- **修订号 (Patch)** - 向后兼容的问题修正

### 发布周期

- **主版本** - 每季度发布
- **次版本** - 每月发布
- **修订版** - 按需发布

### 支持政策

- **当前版本** - 完整支持
- **上一版本** - 关键安全修复
- **更早版本** - 停止支持
