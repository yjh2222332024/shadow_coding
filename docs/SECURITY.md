# 安全政策

## 报告安全漏洞

我们非常重视 Shadow_Coding 的安全性。如果你发现任何安全漏洞，请负责任地披露。

### 📧 联系方式

- **Email**: 2857922968@qq.com
- **GitHub**: https://github.com/yjh2222332024/Shadow_Coding/issues (私密 Issue)

### 🚫 不要

- **不要**创建公开 GitHub Issue 报告安全漏洞
- **不要**在公开场合讨论未修复的漏洞
- **不要**利用发现的漏洞进行非法活动

### ✅ 应该

- **立即**报告任何可疑的安全问题
- **提供**详细的复现步骤
- **等待**漏洞被修复后再公开讨论

## 响应流程

1. **确认收到** - 我们会在 48 小时内确认收到报告
2. **评估** - 我们会评估漏洞的严重性和影响范围
3. **修复** - 开发修复方案并进行测试
4. **发布** - 发布安全更新并通知用户
5. **披露** - 在漏洞修复后公开披露详情

## 支持版本

我们只为当前维护的版本提供安全更新。

| 版本 | 支持状态 | 备注 |
|------|---------|------|
| 3.2.x | ✅ 支持 | 当前稳定版 |
| 3.1.x | ⚠️ 维护 | 仅关键安全修复 |
| 3.0.x | ❌ 停止支持 | 请升级到新版本 |
| 2.x | ❌ 停止支持 | 请升级到新版本 |

## 安全最佳实践

### 使用 Shadow_Coding 时

1. **定期更新** - 保持使用最新版本
2. **审查日志** - 定期检查安全日志
3. **最小权限** - 仅授予必要的权限
4. **备份代码** - 定期备份重要代码

### 贡献代码时

1. **依赖审查** - 仔细审查新增依赖
2. **输入验证** - 始终验证外部输入
3. **安全测试** - 运行所有安全测试
4. **代码审查** - 所有 PR 都需要审查
